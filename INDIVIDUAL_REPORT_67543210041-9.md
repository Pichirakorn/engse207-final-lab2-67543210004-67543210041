## ขอบเขตงานที่รับผิดชอบ

ผมรับผิดชอบในส่วนของการทดสอบระบบ (testing) และช่วยทำ README กับเก็บภาพ screenshots ของระบบ
โดยจะเน้นไปที่การเช็คว่า API แต่ละตัวใช้งานได้จริง และ service ต่าง ๆ เชื่อมกันถูกต้อง

# สิ่งที่ได้ดำเนินการด้วยตนเอง

- ลองเรียก API ต่าง ๆ เช่น register, login, create task, get task ด้วย curl
- ตรวจสอบว่า response ที่ได้ถูกต้องหรือไม่
- ทดสอบการทำงานผ่าน API Gateway ว่าสามารถเรียกไปแต่ละ service ได้
- ช่วยเขียน README อธิบายการใช้งาน วิธีรัน และโครงสร้างระบบ
- แคปหน้าจอการทำงานของระบบบน Railway และตอนทดสอบ API
- เช็ค URL และ environment variables ว่าตั้งค่าถูกต้องไหม

# ปัญหาที่พบและวิธีการแก้ไข

- ปัญหา 1: เข้า URL แล้วขึ้น Route not found ตอนแรกคิดว่าระบบพัง แต่จริง ๆ คือเรียก path ผิด


- ปัญหา 2: บางครั้งเรียก API แล้วใช้ไม่ได้ สาเหตุคือ URL ของ service ผิด หรือ config ยังไม่ตรง แก้โดยเช็ค environment variables แล้วลองใหม่

# สิ่งที่ได้เรียนรู้จากงานนี้

- เข้าใจมากขึ้นว่า microservices คืออะไร และแต่ละ service ทำงานแยกกันยังไง
- รู้ว่า API Gateway ช่วยรวม endpoint ให้ใช้ง่ายขึ้น
- ได้ลองใช้ curl ทดสอบ API จริง
- เข้าใจ flow ของ login และการใช้ token (JWT)
- ได้ฝึกเขียน README และอธิบายระบบ
# แนวทางการพัฒนาต่อไปใน Set 2
- อยากเพิ่ม logging ให้ดู error ได้ง่ายขึ้น

- อาจทำ frontend ให้ใช้งานง่ายขึ้น ไม่ต้องใช้ curl อย่างเดียว
